# SuperT 当前板载配置

## 1. 当前工程到底用了哪块板

当前 `SuperT` 默认构建实际启用的是：

- 硬件基板：`T5AI_BOARD`

当前默认未启用的是：

- `T5AI_BOARD_EYES`
- `T5AI_BOARD_EVB`
- `T5AI_BOARD_ROBOT`
- `T5AI_BOARD_DESKTOP`
- `Ubuntu`

也就是说，当前工程不是“同时启用了多块硬件板”。

## 2. 为什么看起来又像用了 EYES 板

这是因为当前默认配置里启用了：

- `CONFIG_SUPERT_USE_EYES_UI = y`

这个配置最终会在代码中表现为：

- `SUPERT_USE_EYES_UI = 1`

因此当前标准板会复用：

- `T5AI_BOARD_EYES/ui/eyes_app.c`
- `T5AI_BOARD_EYES/ui/eyes/` 下的表情资源

所以当前工程的真实关系应理解为：

- 硬件板级配置：`T5AI_BOARD`
- UI 变体资源：`T5AI_BOARD_EYES`

这属于“标准板复用 EYES UI 资源”，不是“EVB/ROBOT/DESKTOP 这些板的 GPIO 也一起生效了”。

## 3. 当前实际生效的板级链路

当前构建链路可以简化为：

1. `CONFIG_T5AI_BOARD=y`
2. 编译 `T5AI_BOARD/tuya_device_board.c`
3. `tuya_device_cfg.h` 通过 `tuya_device_board.h` 取标准板引脚和显示参数
4. 显示分发时，由 `T5AI_BOARD + CONFIG_SUPERT_USE_EYES_UI=y` 选择 `eyes` UI

所以：

- 硬件初始化来自 `T5AI_BOARD`
- UI 入口默认走 `eyes`

## 4. 当前默认硬件参数

### 4.1 按键和电源相关

- 音频唤醒键：`P19`
- 深睡唤醒定义：`P45`
- 喇叭使能：`P20`
- 网络/功能键：`P18`

### 4.2 电池相关

- 电池功能：开启
- 充电检测：`P21`
- 电量采样：`P28`

### 4.3 显示相关

- LCD 驱动：`spi_gc9a01`
- 分辨率：`240 x 240`
- 旋转：`0`
- SPI CLK：`P14`
- SPI CS：`P15`
- SPI MOSI：`P16`
- LCD DC：`P9`
- LCD RESET：`P17`
- LCD 背光：`P31`

### 4.4 摄像头相关

虽然 `T5AI_BOARD` 的板级头文件里保留了摄像头默认参数，但当前构建配置中：

- `CONFIG_ENABLE_TUYA_CAMERA` 未开启

因此当前默认编译结果里，摄像头功能并未启用。

## 5. 当前默认功能配置

- 使用板级音频输入
- 使用板级音频输出
- UI 已启用
- 默认对话模式为 `FREE`
- 默认板级 UI 走 `EYES` 资源风格
- 默认低功耗模式为深睡

## 6. 对“是不是混用了其他板载”的最终判断

### 是什么

- `T5AI_BOARD` 基板
- `T5AI_BOARD_EYES` UI 资源复用

### 不是什么

- 不是同时启用了 `EVB`
- 不是同时启用了 `ROBOT`
- 不是同时启用了 `DESKTOP`
- 不是同时启用了 `Ubuntu`

## 7. 如果你想把它彻底整理成“一眼能看懂的单板配置”

当前最推荐的理解和维护方式是：

### 方案 A：保留现状，但明确命名

把当前项目理解为：

- `T5AI_BOARD` 标准硬件板
- `T5AI_BOARD + CONFIG_SUPERT_USE_EYES_UI=y`

这是当前工程最接近真实情况的描述。

### 方案 B：后续继续做代码整理

现在这一步已经先把“硬件板型”和“UI 变体”拆成了两层配置：

- 第一层：`CONFIG_T5AI_BOARD`
- 第二层：`CONFIG_SUPERT_USE_EYES_UI`

如果你后面还想继续清理，我建议再把 UI 变体扩展成更明确的枚举化配置：

- 第一层：硬件板型
  - 标准板 / EVB / ROBOT / DESKTOP
- 第二层：UI 变体
  - wechat / eyes / robot / desktop

这样以后就不会再出现“明明是标准板，却在编译 EYES 资源，看起来像混板”的理解成本。

## 8. 当前建议

结合当前代码和编译配置，建议你在对外描述或团队内部说明时直接写成：

> 当前项目默认构建为 `T5AI_BOARD` 硬件基板，默认复用 `T5AI_BOARD_EYES` 的表情 UI 资源，未启用 EVB、ROBOT、DESKTOP 等其他硬件板型。
